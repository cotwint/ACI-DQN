1. PROJECT OVERVIEW
================================================================================

  Problem:  Datacenter server scheduling under heterogeneous tasks (P1/P2/P3)
            and time-of-use electricity pricing. Every 15 minutes the scheduler
            decides how many servers to keep active. Trade-off: too few ->
            SLA violations (costly); too many -> wasted electricity.

  Solution: ACI-DQN — a DQN agent whose state is augmented with online
            conformal prediction intervals (ACI). The intervals tell the
            Q-network "how uncertain the future is", enabling robust
            decisions under distribution shift.

  Nine main methods compared:
    1. Fixed                    — constant server count
    2. Queue-Greedy             — serve current backlog, ignore price
    3. Price-Aware Greedy       — defer P3 when electricity is expensive
    4. Forecast-Greedy          — rolling-mean point forecast + greedy planning
    5. Conformal-Greedy         — ACI interval forecast + greedy planning
    6. DQN                      — standard DQN (17-dim state, no uncertainty)
    7. Forecast-DQN             — DQN + K point-forecast features (20-dim)
    8. Static-Conformal-DQN     — DQN + 2K fixed split-conformal features (23-dim)
    9. ACI-DQN                  — DQN + 2K ACI online-adaptive features (PROPOSED, 23-dim)

  Appendix method:
    Shielded DtACI-DQN          — DQN + DtACI ensemble intervals + safety shield

3. SCENARIO SYSTEM (config isolation)
================================================================================

  Design principle: Base config.yaml holds system defaults (Nmax=120, ramp_limit=25,
  switch_cost=0.10). Each E0-E4 scenario YAML overlays capacity constraints and
  workload enhancements. Deep-copy is used so the base config is NEVER mutated.

  Scenario YAML structure (e.g., E2_distribution_shift.yaml):
    scenario: {id: E2, name: "Distribution Shift"}
    server: {Nmax: 75, ramp_limit: 15}
    power: {switch_cost: 0.30}
    workload_enhancement:           # shared root enhancements
      day_multiplier: {mu: 0.0, sigma: 0.3}
      autocorr_noise: {rho: 0.7, sigma: 0.15}
    phases:
      train: {}                     # empty = inherit from root
      calibration: {}               # empty = inherit from root
      test:                         # test: additional overrides
        workload_enhancement:
          day_multiplier: {mu: 0.2, sigma: 0.1}
          priority_mix_shift:
            enabled: true
            mode: redistribute
            p1_share_factor: 2.0
            incident_hours: [10, 16]

  Key functions in src/scenarios.py:
    load_scenario_overlay(id)    → loads YAML from configs/scenarios/{id}.yaml
    build_scenario_config(id, base_cfg) → deep-copies base, deep-merges overlay
    get_phase_config(scenario_cfg, phase) → merges scenario root + phases.{phase}
    apply_scenario_to_env(env, phase_cfg) → env.set_price_curve(), env.set_workload_params()

  Deep merge is recursive — E3 test adds clustered_burst ON TOP of root
  day_multiplier+autocorr_noise, not replacing the whole block.

4. DATA PIPELINE
================================================================================

  Stage A — Preprocessing (data_preprocess.py)
  --------------------------------------------
  load_raw_csv("区域负荷2020-2023数据.csv")
    → detects Chinese column names (时间, 有功功率)
    → returns DataFrame[timestamp, regional_load_kw]

  clean_and_resample()
    → drop duplicates, reindex to 15-min grid, linear interpolate (limit=4)

  keep_complete_days()
    → keep only days with all 96 slots non-null
    → output: processed_load.csv

  chronological_split()
    → train: 60% (earliest), calibration: 20%, test: 20% (latest)

  Stage B — Building env ( _common.py → build_env_and_splits() )
  ---------------------------------------------------------------
  load_processed() → load_df, split_df
  day_matrix()     → raw_load:  (D, 96) kW per slot
  normalised_day_matrix() → norm_load: (D, 96) in [0, 1]

  Workload generation path:
    IF workload_enhancement params set on env:
      generate_day_enhanced(norm_load[day], cfg, seed, enhancement)
    ELSE:
      generate_day(norm_load[day], cfg, seed)

  Trace path:
    load_trace_workloads(trace_dir) → Dict[int, DayWorkload]
    DataCenterEnv(..., external_workloads=workloads)
    → env.reset(day) looks up external_workloads[day]

  Stage C — Enhanced Workload Generation (workload_generator.py)
  ---------------------------------------------------------------
  Deterministic RNG hierarchy (7 sub-seeds from unified seed):
    rng = np.random.default_rng(seed)
    sub_seeds = rng.integers(0, 2**31-1, size=7)
    # 0: day_multiplier, 1: autocorr_noise, 2: priority_mix_shift,
    # 3: burst_state, 4: burst_multiplier,
    # 5: Poisson arrivals, 6: work sampling

  Enhancement functions (each takes explicit seed):
    apply_day_multiplier(lam, params, seed)      → LogNormal(mu, sigma) scalar
    apply_autocorr_noise(lam, params, seed)      → AR(1): z_t = ρ·z_{t-1} + ε_t
    apply_priority_mix_shift(lam, params)         → redistribute or amplify modes
    apply_clustered_burst(lam, params, seed)      → Markov-chain burst state machine

  Trace guard:
    workload.source == "trace" + synthetic calibration → NotImplementedError

5. ENVIRONMENT — DataCenterEnv (src/datacenter_env.py)
================================================================================

  Constructor
  -----------
  DataCenterEnv(cfg, day_load_matrix, day_norm_matrix, base_seed,
                external_workloads=None)

  Public mutation API (ONLY way to reconfigure env after construction):
    env.set_price_curve(curve: np.ndarray)       — override electricity price
    env.set_workload_params(params: dict)         — enable enhanced workload gen

  Strict Action-Index API
  -----------------------
  Discrete action space: 0 .. action_bins-1 (default 21 bins)

  env.step(action) REQUIRES action ∈ [0, bins-1]. Passing a server count
  (e.g., 68) will raise ValueError. Use n_to_action() and action_to_n()
  for conversions:

    action_index  →  action_to_n()  →  server_count  →  shield()  →
    server_count  →  n_to_action()  →  action_index  →  env.step()

  State Space (17 dims, normalized)
  ---------------------------------
    dim  0-2:  Q_k / 500                (queue length per priority)
    dim  3-5:  B_k / 500                (backlog work per priority)
    dim  6-8:  near_deadline_k / D_k    (tasks within 25% of deadline)
    dim 9-11:  min_slack_k / D_k        (minimum remaining slack per priority)
    dim   12:  price                    (electricity price)
    dim   13:  x(t)                     (normalised load in [0,1])
    dim   14:  n_prev / Nmax            (previous active server count)
    dim   15:  sin(2π·t/T)              (time-of-day encoding)
    dim   16:  cos(2π·t/T)              (time-of-day encoding)

  With augmenters:
    Forecast-DQN:            17 + K = 20 dims  (K point-forecast features)
    Static-Conformal-DQN:    17 + 2K = 23 dims (2K fixed interval features)
    ACI-DQN:                 17 + 2K = 23 dims (2K ACI online intervals)

    Unified feature order: [upper_1, upper_2, upper_3, lower_1, lower_2, lower_3]

  Dynamics (env.step(action_index))
  ---------------------------------
  1. n_active = Nmin + (Nmax-Nmin) * action_index / (bins-1)
  2. Apply ramp limit: clip(n_active, n_prev-ramp_limit, n_prev+ramp_limit)
  3. Arrive new tasks (from DayWorkload.tasks_per_slot[k][t])
  4. Dispatch (strict priority + EDF within priority):
       For k in 0..2 (P1 first):
         Sort queue[k] by deadline ascending
         Serve tasks until capacity exhausted
         Record completed, violations (delay > deadline), delay_sum
  5. Compute costs (elec, qos, switch, total)
  6. Reward = -Σ(w_i * cost_i / scale_i)

6. RL COMPONENTS
================================================================================

  DQNAgent (src/rl/dqn_agent.py)
  ------------------------------
  QNetwork: MLP [in_dim, 128, 128, out_dim] with ReLU
  ReplayBuffer: deque with uniform random sampling
  Training: smooth L1 loss, Adam optimizer, target network
  Epsilon: 1.0 → 0.05, decay per episode

  StateAugmenter Protocol (src/rl/augmenters.py)
  ----------------------------------------------
  Each augmenter provides:
    reset(env, day_index)           — per-episode init
    augment(state, env) → array     — extend state with extra features
    shield(n_servers, state, env) → int — modify action (server count → server count)
    on_step(env, info, a_pre, a_post) — per-step callback
    metrics() → dict                — diagnostic info
    warm_up_from_calibration(y_hat, y_cal) — optional, for conformal augmenters

  IdentityAugmenter (plain DQN):
    augment → returns state unchanged (17 dims)
    shield  → returns n_servers unchanged

  ForecastAugmenter (Forecast-DQN):
    Uses same rolling-mean forecaster as ACI-DQN
    augment → state (17) + K point-forecast features → 20 dims
    No conformal intervals, no shield

  StaticConformalAugmenter (Static-Conformal-DQN):
    Uses same forecaster + SplitConformal fitted ONCE in warm_up_from_calibration()
    augment → state (17) + 2K fixed interval features → 23 dims
    Feature order: [upper_1, upper_2, upper_3, lower_1, lower_2, lower_3]
    No online update in on_step()

  ConformalAugmenter (ACI-DQN):
    Uses same forecaster + ACI learner per priority
    augment → state (17) + 2K ACI online interval features → 23 dims
    on_step → calls ACI learner.update() with actual arrivals
    shield  → no-op when use_shield=False (main ACI-DQN)

  Training Loop (train_agent in train_dqn.py)
  -------------------------------------------
  For each episode:
    1. Sample a training day
    2. rollout_episode(env, agent, augmenter, day, train=True):
         reset env → while not done:
           a_raw = agent.select_action(s_aug)     # ε-greedy, action index
           a_n   = action_to_n(a_raw)             # → server count
           a_safe = augmenter.shield(a_n, ...)    # → possibly modified
           a_safe_idx = n_to_action(a_safe)        # → back to action index
           next_state, reward, done, info = env.step(a_safe_idx)
           augment state, push to buffer, train step
    3. agent.decay_epsilon()

  Evaluation (evaluate in train_dqn.py)
  -------------------------------------
  Same as rollout but greedy=True, deterministic seeds per day.
  All methods see identical task traces per day for fair comparison.
  Optional: collect_aci_diagnostics=True records per-slot coverage,
  alpha, interval bounds, residual quantile → aci_diagnostics.csv.

7. POLICY PROTOCOL (heuristic baselines)
================================================================================

  All policies follow unified interface:
    reset()                        — per-episode init (no-op for simple policies)
    act(state, env) → int          — return server count
    on_step(env, info)             — per-step callback (no-op for simple policies)
    metrics() → dict               — diagnostic info
    warm_up_from_calibration(y_hat, y_cal) — optional, for conformal policies

  FixedPolicy:       always returns (Nmin+Nmax)//2
  QueueGreedyPolicy: effective_work formula, ignores price
  PriceAwareGreedy:  same + defers P3 when price is high
  ForecastGreedy:    rolling-mean H-step forecast + capacity planning
  ConformalGreedy:   ACI upper bounds + same capacity planning formula

8. CONFORMAL PREDICTION LAYER
================================================================================

  Shared base forecaster (all methods use the same one):
    RollingMean(window=8) → ŷ_k(t) = mean of past arrival counts

  ACI (src/conformal/aci.py)
  --------------------------
  State: α_t (effective miscoverage rate), residuals buffer

  interval(y_hat):
    q_level = 1 - α_t
    q = quantile(residuals, q_level, method="higher")
    return [y_hat - q, y_hat + q]

  update(y_hat, y_true, lo, hi):
    covered = int(lo <= y_true <= hi)
    err = 1 - covered
    α_{t+1} = clip(α_t + η * (α_target - err), α_min, α_max)

  DtACI (src/conformal/dtaci.py) — appendix only
  ----------------------------------------------
  Ensemble of M=5 ACI experts with different η. Mixture weights via
  exponential weighting on pinball loss.

9. MULTI-SEED EVALUATION
================================================================================

  Experiment loop (main.py):
    For each scenario_seed in explicit list:
      build scenario config, get phase configs, apply to env
      For each method:
        IF heuristic:
          one run per scenario_seed, training_seed=None
        ELSE (RL):
          for each training_seed:
            train agent → evaluate on test days

  Output CSVs:
    daily_results.csv       — per-day, per-method, per-seed metrics
    experiment_summary.csv  — mean ± std ± 95% CI via scipy t-dist
    paired_tests.csv        — paired t-tests vs reference methods
    aci_diagnostics.csv     — per-slot ACI coverage/alpha/interval width

  Paired test logic (src/evaluation/metrics.py):
    RL-vs-RL:   pair by scenario_seed + training_seed + day
    Heuristic-vs-RL: pair by scenario_seed + day (no training_seed)

10. REAL-TRACE INTERFACE (src/trace_loader.py)
================================================================================

  load_trace_workloads(trace_dir, cfg, day_dates) → Dict[int, DayWorkload]

  To switch from synthetic to real traces:
    1. Set workload.source = "trace" in config.yaml
    2. Place trace CSVs in trace_dir
    3. Run as normal — _common.py auto-detects and loads

  Trace guard: if workload.source == "trace" and calibration is asked to
  generate synthetic arrivals → NotImplementedError. Real-trace calibration
  must use actual trace arrival data.
